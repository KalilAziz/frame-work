Prototypes

O Js é baseado em protótipos para passar propriedades e métodos de um objeto para outro.

Definição de protótipo:

Protótipo é o termo usado para se referir ao que foi criado pela primeira vez, servindo de modelo ou molde oara futuras produções.

Todos os objetos tem uma referêncai interna para um protótipo (_proto_, que pode ser vista em console.dir) que vem da propriedade prototypes da função construtora que foi usada para criá-lo. Quando tentamos acessar um mebro de um objeto, primeiro o motor do JS vai tentar encontrar este membro no proprio objeto e depois a cadeia de protótipos é usada até o topo (null) até encontrar (ou não) tal membro.

Exemplo: Para uma construtora, ter o método nomeCompleto é tranquilo, mas se ela tiver que ser usada milhares de vezes, milhares de vezes o método nomeCompleto vai ter que ser criado, ou seja, teremos milhares de códigos iguais. Então com o Objetivo de economizar recursos computacionais, podemos colocar esse método em outro lugar, onde ele será utilizado sempre uma vez, não importa da onde.

Normalmente, iremos usar o prototype em uma constructor function, e para que possamos usar o prototype, precisamos apenas que colocar o nome da constructor function, notação de ponto, prototype, notação de ponto novamente, e o nome do método, onde iremos atribuir uma function anônima. O prototype vai literalmente inserir esse método na nossa constructor function. Vale lembrar que se o objeto tiver um método com o nome igual a um prototype existente, ele irá sobreescrever o prototype.
