Funções imediatas (IIFE)

Para não poluir o escopo global anteriormente, sempre declaramos usando uma função específica e só faziamos a chamada da função no final, para que todo o código seja executada, mas dessa forma não é a maneira mais correta, pois já da função ter um nome, o escopo global já será poluido, mesmo que muito pouco. E para resolver esse problema, iremos colocar dois pares de abre e fecha parenteses, onde o primeiro parenseteses iremos colocar uma função aninima que ireá segurar todos os códigos do nosso sistema. O ultimo par de parenteses, poderemos passar valores, para que possamos resgatar nos parâmetros da função anonima no primeiro par de parenteses e usar em todo o sistema.
