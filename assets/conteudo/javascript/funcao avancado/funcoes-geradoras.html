Funções geradoras

As funções geradoras tem uma capacidade de quase pausar o código em um lugar especial. Vamos supor que uma função tem três valores, onde que cada vez que chamar a função, iremos querer que ela entregue um dos três valores, podendo ser bom para a perfómace do código em algumas situações. A diferença na declaração dela é que usamos um asterístico junto com function. Os diferentes valores que essa function retorna, temos que colocar o yield, onde a cada vez que chamar, irá desder o yield.

E para chamar os yield das funções geradoras, temos que chamar elas normalmente, juntamente com .next(), onde ela retornará os valores em forma de objetos, onde terá dois atributos, o value, que carrega o que o yield retorne e o done: que será false até que o ultimo yield da função seja executado, quando todos os yield terem sido chamados e a função for chamado novamento, o done será true.

Para que possamos especificar qual dos valores queremos pegar, seja o value ou o done, temos que usar .next() e marcação de ponto, onde iremos definir qual valores iremos querer.

Podemos utilizar o for para pegar os valores automaticamente.

Gerador infinito

Uma das grandes vantagens das funções geradoras é que podemos executar algo diferente toda vez que chamarmos a função, mas o problema é que temos uma quantidade limitada de vezes que possamos fazer essa chamada, ou seja, se definimos apenas 4 yield, nossa função só poderá ser chamada 4 vezes, qualquer chamada além dessa, não retornará nenhum valor. Se acontecer alguma situação que precisamos chamar a função geradora um número desconhecido de vezes, teremos que fazer uma função geradora infinita, e para isso, teremos que colcoar um laço de repetição dentro da função geradora, onde os yield estarão declarados lá dentro. Primeiramente temos que declarar a função geradora, e dentro dela colocaremos um while(){}. Antes, teríamos que colocar uma condição dentro dos parenteses, para que os códigos dentro das chaves sejam executados, mas dessa vez, já iremos definir como true dentro dos parenteses, para que eles seja sempre executado, e dentro das chaves, iremos colcoar os yield.

Delegar tarefas para outro gerador

Podemos fazer um trabalho de delegação de uma tarefa de uma função geradora para outra. Para que possamos fazer essa delegação é bem simples, precisamos apenas colocar o chamado da função geradora que desejamos delegar a tarefa em um dos yield da nossa função geradora principal, onde aquele yield irá se repetir enquando toda a função geradora a qual foi delegada uma tarefa ter terminado todos os seus yield, onde posteriromente, o restante dos yield da função geradora principal continuarão ser executados normalmente até terminarem. O único detalhe que temos que prestar atenção é que quando formos fazer a chamada da função que quereros delegar uma tarefa, o yield fa função principal terá que ter um asterístico.

Qualquer tipo de dados é possível

Os yield de uma função geradora pode ser usado para executar qualquer tipo de dado ou estrutura de dados, seja Number, String, Booleanos, Arrays, Objetos, Function. Mas precisamos ter cuidado, pois em casos de função, se colocarmos return, ele vai encerrar a função e não conseguiremos pegar mais nenhum yield abaixo, então temos que evitar o uso do retorn.
