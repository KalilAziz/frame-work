Tradicional

A maneira mais tradicional de declarar uma function é apenas usando a palavra function, o nome da function juntamente com abre e fecha parenteses, e posteriromente colocamos abre e fecha chaves que será o corpo da função, onde colocamos o bloco de códigos. Para chamar a função, basta cololocar o nome da função e abre e fecha parenteses.

See the Pen 068.4 - Maniras de declarar funções by KalilAziz (@kalilaziz) on CodePen.

Funtion expression

Uma das maneiras de declarar uma função é a funtion expression, onde podemos criar uma variável e colocar a função como atribuição dessa variável. E para chamar a função dentro da variável, basta chamar a variável como chamaríamos uma função, sendo necessário colocar o nome da variável e abre e fecha parenteses. Essa característica é muito desejada, pois podemos passar a função como argumento e usa-la pelos parâmetros de outra função. Vale lembrar que podemos usar a function da mesma maneira sem a utilização de atribuir a uma variável, se tornando uma function anônima.

See the Pen 068 - Maniras de declarar funções by KalilAziz (@kalilaziz) on CodePen.

Arrow function

A Arrow funciton é uma nova maneira de declarar uma função, sendo a mais moderna possível. Para que possamos utilizar a arrow funciton, na maioria das vezes iremos declara uma variável para que possamos atribuir a arrow funciton, onde teremos que colcoar abre e fecha parenteses, que serão os parâmetros e por fim, colocamos => {}, onde dentro dos parenteses serão os blocos de códigos. Lembrando que se não atribuirmos a arrow function a uma variável, ela será uma function anônima.

Observações importantes

As funcçóes tem uma característica denomidade de function hoisting, ou seja, podemos chamar uma função antes ou depois dela ser declarada                                                                          

As funções são First-Class Objects (objetos de primeira classe), ou seja, a função pode ser tratada com JS como dados
