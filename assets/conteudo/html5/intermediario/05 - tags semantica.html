Tags Semânticas

Com a chegada do HTML5, surgiu algumas tags semânticas que facilitam os motores de busca o google, sendo elas:

Header

<header> </header>: O header é utilizado para representar o cabeçalho de um documento ou seção declarado no HTML. Nele podemos inserir elementos de <h1> a <h6>, até elementos para representar imagens, parágrafos ou mesmo listas de navegação.

See the Pen Header by KalilAziz (@kalilaziz) on CodePen.

Nav

<nav> </nav>: O elemento nav é utilizado quando precisamos representar um agrupamento de links de navegação, que, por sua vez, são criados com os elementos <ul>, <li> e <a>.

See the Pen Nav by KalilAziz (@kalilaziz) on CodePen.

Footer

<footer> </footer>: O elemento footer representa um rodapé de um documento, como a área presente no final de uma página web. Normalmente é utilizado para descrever informações de autoria, como nome e contato do autor, e data de criação do conteúdo.

See the Pen Footer by KalilAziz (@kalilaziz) on CodePen.

Main

<main> </main>: O elemento main especifica o conteúdo principal e, consequentemente, de maior relevância dentro da página. Para ser considerada bem construída, uma página deve apresentar apenas um conteúdo principal.

See the Pen Main by KalilAziz (@kalilaziz) on CodePen.

Section

<section> </section>: O elemento section representa uma seção dentro de um documento e geralmente contém um título, o qual é definido por meio de um dos elementos entre <h1> a <h6>. Podemos utilizar o section, por exemplo, para descrever as seções/tópicos de um documento.

See the Pen Section by KalilAziz (@kalilaziz) on CodePen.

Article

<article> </article>: Utilizamos o elemento article quando precisamos declarar um conteúdo que não precisa de outro para fazer sentido em um documento HTML, por exemplo, um artigo em um blog. É recomendado identificar cada article com um título.

See the Pen Article by KalilAziz (@kalilaziz) on CodePen.

Aside

<aside> </aside>: O elemento aside é utilizado quando precisamos criar um conteúdo de apoio/adicional ao conteúdo principal. Por exemplo, ao falar de HTML semântico, podemos indicar ao leitor outros conteúdos sobre a linguagem HTML como sugestão de leitura complementar.

See the Pen Aside by KalilAziz (@kalilaziz) on CodePen.
